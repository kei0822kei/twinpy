twinpy.common.kpoints.get_mesh_offset_from_direct_lattice(lattice: numpy.array, interval: Optional[float] = None, mesh: Optional[list] = None, include_two_pi: bool = True) → dict[source]¶

Get kpoints mesh and offset from input lattice and interval.

Parameters

  • lattice (np.array) – lattice matrix

  • interval (float) – grid interval

  • mesh (list) – mesh

  • include_two_pi (bool) – if True, include 2 * pi

Returns

containing abc norms, mesh, offset

Return type

dict

Raises

  • ValueError – both mesh and interval are not specified

  • ValueError – both mesh and interval are specified

Note

Please input ‘interval’ or ‘mesh’, not both. If the angles of input lattice is (90., 90., 120.), offset is set (0., 0., 0.5) and mesh is set (odd, odd, even or 1). If even becomes zero, fix to one. Otherwise, set (0.5, 0.5, 0.5) and mesh is set (even or 1, even or 1, even or 1). If you use this function, it is better to input STANDARDIZED CELL.

Transform float to fractional (str object).

Parameters

  • var – Input value.

  • accuracy – Threshold of whether two values are the same of not. In the case of ‘accuracy=3’, check the third decimal place.

  • denominator – If ‘denominator=10’, check from 1/2 to 12/13.

Returns

Fractional value.

Return type

str

Raises

RuntimeError – Could not find fractional representation of input ‘var’.

Examples

>>> float2frac(-1.333333, 3)
  '-4/3'
